동영상 콘텐츠 & 대구 브로셔 자동화를 위한 AI 에이전트 설계 (2025년 10월)
Make.com AI Agent 최신 UI(2025년 10월 기준) 설정

Make.com의 최신 AI Agent 기능을 활용해 콘텐츠 제작 워크플로우에 특화된 전용 에이전트를 구성합니다. 핵심 단계와 설정은 아래와 같습니다.

AI 제공자 & 모델: Make.com에서 새 AI Agent를 만들고 OpenAI GPT-4/ChatGPT-5 모델( Make의 AI provider 연결을 통해)에 연결합니다. 이것이 에이전트의 추론 엔진이 됩니다. 고품질 생성을 위해 GPT-4/GPT-5를 선택합니다.

시스템 프롬프트: 에이전트의 역할·목적·제약을 명확히 정의한 시스템 프롬프트를 설정합니다. 예시: “당신은 짧고 재미있는 동영상 대본과 여행 브로셔 콘텐츠를 만드는 AI 어시스턴트입니다. 데이터 수집, 스크립트 생성, TTS 오디오 생성, 이미지 생성, 결과 조립을 위해 정의된 도구들을 사용하세요. 주부 대상 유머/공감 톤 또는 여행 콘텐츠엔 밝고 매력적인 톤을 사용하세요.” 이 프롬프트는 일관된 페르소나와 지시를 제공합니다.

컨텍스트(지식 베이스): Context 기능으로 에이전트가 참조할 레퍼런스 파일을 업로드합니다. 예: 유머 스타일 가이드, 주부 에피소드 예시 라이브러리, 대구 관광 정보. 파일은 최대 20개(각 ≤20MB, txt/PDF/docx 등)까지 추가 가능하며, 장기 메모리처럼 사실·스타일 가이드에 접근할 수 있습니다.

MCP 통합: Make는 에이전트에 MCP(Model-Context-Protocol) 도구 연결을 지원합니다. 에이전트 설정의 MCP 섹션에서 사용 가능한 MCP 서버를 연결하여 외부 도구 능력을 부여합니다. 웹검색/특화 DB 등 MCP가 제공되면 이상적입니다. 연결 후 해당 서버의 툴(API) 을 개별 토글로 활성화하세요. (참고: MCP 없이도 Make 내장 모듈/시나리오만으로 충분히 구축 가능)

도구(시나리오 & 모듈): 에이전트의 Tools 섹션에 워크플로우에 필요한 툴을 추가합니다. 2025년 10월 업데이트로 도구 유형은 두 가지입니다.

모듈 툴(Module Tools): 단일 스텝 모듈 액션(예: 한 번의 OpenAI 호출, 한 번의 Google TTS 호출). 모듈을 툴로 추가하면 Make가 입력/출력을 갖춘 시나리오를 자동 생성합니다. 간단 작업에 적합(예: “Google Sheets: Add Row”, “OpenAI: Create Image”, “Google TTS: Synthesize”).

시나리오 툴(Scenario Tools): 다중 스텝 복합 작업용. 커스텀 시나리오를 만들고 입·출력 필드를 수동 정의해 에이전트가 호출 가능하도록 합니다. 각 시나리오는 On demand(주문형) 으로 활성화해야 에이전트가 임의 시점에 호출할 수 있습니다. 구성 시:

시나리오 입력 필드/출력 필드를 이름·설명·타입과 함께 정의(예: “GenerateVideoScript(topic:string) → script:text”).

비주얼 플로우에서 입력을 모듈 매핑, 최종 산출을 정의.

저장 후 활성화하고 에이전트 도구 목록에 추가.

모듈 vs 시나리오 선택: 단일 모듈로 가능한 업무는 모듈 툴이 간편합니다(수동 I/O 매핑 불필요). 예: TTS. 반면 검색→생성→후처리 같이 다단계면 시나리오 툴이 적합합니다. 문서 권장처럼 시나리오가 더 손이 가지만 유연합니다. 우리는 원자 작업은 모듈, 오케스트레이션은 시나리오로 혼용합니다. 필요하면 모듈 툴을 “언락”→풀 시나리오로 변환도 가능.

테스트 & 튜닝: 에이전트 설정의 Testing chat에서 샘플 지시로 도구 호출과 결과를 확인·개선합니다. 툴 이름/설명을 명확히 하여 에이전트가 올바른 타이밍에 툴을 고르도록 하고, 시스템 프롬프트는 간결하게 유지하며 필요한 경우 점진 보강합니다.

이제 두 가지 핵심 목표에 맞춰 자동화 워크플로우를 설계합니다: (A) 트렌딩 스토리 기반 영상 생성, (B) 대구 관광 브로셔 생성.

자동화 동영상 콘텐츠 생성 워크플로우

간단한 아이디어 입력만으로 내레이션 영상을 자동 제작합니다. 단계별 도구와 Make 구현은 다음과 같습니다.

트렌딩 아이디어 발굴: 입력: 한국 주부 공감/유머, 일본인 아내-한국 남편 에피소드 등 공감도 높은 스토리.

트렌드 리서치 툴: Google Trends·Naver Data Lab로 거시 키워드, 실제 이야기는 커뮤니티(한국·일본) 가 핵심. FM코리아·루리웹 유머, 네이버 카페(맘/다문화), 일본 포럼에서 조회/공감 많은 글을 수집. YouTube/Instagram 트렌딩 해시태그 탐색도 병행. 스크래핑 자동화가 어렵다면 반자동: 팀이 유망 스토리 링크/요약을 Google Sheet에 기록. 이 시트가 에이전트의 아이디어 DB가 됩니다.

Google Sheets 연동: “Story_ID / Title / Summary / SourceLink / PopularityScore” 컬럼. Make의 Google Sheets 모듈(“Search Rows”, “Get a Cell”)로 미사용·고점수 아이디어를 선택. 완전 허구 대신 실제 인기 스토리 기반으로 생성해 성과 가능성을 높입니다.

스크립트 생성(Text): 선택 스토리를 영상 내레이션용 대본으로 작성.

ChatGPT 툴: OpenAI Chat( GPT-4/5 ) 모듈로 “요약→대본” 프롬프트. 예: “[요약]을 1인칭, 대화체, 주부 대상 유머/공감 톤으로 대본화. 낭독 3–4분 분량.” 시스템/컨텍스트 파일로 스타일 가이드를 주입. 모듈 툴로 등록해 에이전트가 스토리 컨텍스트를 넘기면 스크립트를 반환.

산출: 내레이션 스크립트(한글).

TTS 오디오 생성: 스크립트를 음성으로 변환.

TTS 서비스: Google Cloud TTS(또는 Naver Clova/Polly). Make에서 모듈 툴로 한국어 여성·친화적 보이스 지정, 스크립트→MP3 생성. 파일은 Google Drive 업로드 모듈로 저장·공유 링크 확보.

산출: 오디오 파일(MP3) URL/바이너리.

이미지 생성/수집: 오디오에 맞는 비주얼 준비.

AI 이미지: OpenAI DALL·E 3(또는 Stable Diffusion API)로 일러스트 생성(예: “주방 해프닝을 코믹 카툰 스타일로”). 모듈 툴로 등록해 설명→이미지 URL 반환.

스톡 이미지: Unsplash/Pexels API(HTTP 모듈)로 테마 사진 대체 가능.

기본은 커버 이미지 1–2장 생성(중간 컷 전환으로 시각 흥미 유지).

영상 합성: 오디오+이미지를 MP4로 결합.

영상 합성 도구: 완전 자동화를 위해 클라우드 서비스 활용.

CloudConvert API: 이미지+오디오→MP4 슬라이드쇼 생성(가능한 변환 플로우 구성).

또는 서버리스 FFmpeg(HTTP 웹훅 호출)로 “정지 이미지 + 배경 오디오” 합성(간단한 팬/줌 효과 포함 가능).

저장 & 게시 전 준비: 완성 MP4는 Google Drive 저장 또는 바로 YouTube 업로드(Make YouTube 모듈)로 비공개/등록 업로드.

리포트: Slack/이메일 모듈로 제목/길이/미리보기 링크 보고.

검토 & 수정: 사람이 시청 후 톤/발음 등 수정사항 있으면 텍스트 또는 보이스만 재생성→재합성. 모듈화 덕에 부분 재생성이 용이.

오케스트레이션 설계:

단일 시나리오 툴: StoryVideoGenerator(Idea_ID) 하나가 전 과정을 수행(시트→GPT→TTS→이미지→FFmpeg→링크). 에이전트는 이 툴 한 번 호출로 실행.

다중 툴 에이전틱 운용: getIdea→writeScript→makeAudio→makeImage→assembleVideo를 각각 모듈/시나리오 툴로 나눠, 에이전트가 순차 호출. 재사용성·유연성 ↑. 시작은 단일 시나리오로 단순화, 이후 필요 시 세분화 전환(모듈→시나리오 변환 지원).

실행: “새로 반응 큰 주부 에피소드로 영상 만들어줘”처럼 지시하면 에이전트가 자동으로 검색/대본/TTS/이미지/합성까지 수행, 결과 링크와 요약을 응답으로 제공합니다.

연동 포인트 요약:

Google Sheets: 아이디어 DB/로그

YouTube: 최종 공개 채널(제목·설명·해시태그 자동 생성 가능)

Instagram: 아이디어 리서치/재업로드 활용(릴 업로드는 API 제한 고려)

ChatGPT(GPT-4/5): 대본·카피·번역

트렌딩 스토리 소싱(주부 유머 & 국제결혼)

파이프라인을 채우기 위해 한국/일본 커뮤니티에서 고반응 스토리를 심층 조사했습니다. 목표 장르는 일상 유머, 주부 공감, 문화차 에피소드(일본인 아내-한국인 남편 등)입니다. 조회·추천·댓글이 많은 글을 우선 채택했습니다.

한국 커뮤니티: FM코리아·루리웹 등에서 바이럴된 사례 다수. 예: “일본인 아내와 싸운 한국 남편” 글은 조회 29만+, 추천 약 천, 댓글 수백 개로 대히트(2022). 남편이 아내의 K-POP 굿즈를 버려 부부싸움이 난 사연으로, 유머·문화 포인트가 결합. 이런 군중 검증 스토리를 아이디어 시트에 적재했습니다.
추가 소스:

네이트판/다음카페 주부 일상·감동글(바이럴되면 유머 게시판으로 재확산)

유튜브 사연 채널(인기 사연 주제·댓글 확인)

페이스북 유머 페이지(핫 게시물 큐레이션)

일본 커뮤니티: 일본 시점의 한일커플/주부 이야기도 조사.

Ameba/Hatena 블로그: 한국 생활 에피소드(시부모, 명절 등) 고댓글 글.

YouTube(일본): 日韓カップル 채널의 인기 영상 주제·댓글 분석.

Twitter/Threads: 예) “해외 생활 5년 지나면 한국 남편이 미친다(?)”류 조언 스레드와 논쟁.

Konest(일본문 Daegu/한국문화 사이트): 한국 생활/문화 충돌 기사.

스토리 뱅크 운용: 각 스토리의 핵심 플롯/왜 인기였는지(펀치라인·교훈·공감포인트)를 기록. 에이전트는 이를 입력으로 대본 재작성. 컨텍스트 파일에 원문 요지 묶음을 저장할 수도 있어, “세븐틴 굿즈 사건 이야기로 만들어줘” 지시에 즉시 참조 가능.
톤 관리: 유머는 가볍게, 감동은 공감과 배려로. 공감(共感) 을 일으켜 공유 가치를 높입니다.
트렌드 모니터 자동화:

주기적(주 1회) RSS/HTML 체크(유머 핫 게시글 타이틀·조회수).

Reddit r/korea/r/japan의 국제결혼 스레드(영문이지만 로컬라이즈 소재로 활용).

네이버 다문화 카페(공개글 위주).
모든 결과는 Google Sheet에 인기도 지표와 함께 적재, 자동 아이디어 선택에 활용.

대구 관광 브로셔 설계(자동/반자동)

10쪽 이내의 일본인 타깃 대구 하이라이트 브로셔(K-pop/K-drama, 포토스팟, 로컬 맛집)를 AI로 작성·레이아웃합니다.

일본인이 좋아하는 대구 포인트(리서치):

K-POP 랜드마크: 대구 출신 BTS V(태형), 슈가. 시내 BTS 벽화/아트월 필수 방문지. V의 모교 인근 V Art Wall(별이 빛나는 밤 콘셉트)도 핫 포토스팟.

드라마 촬영지: 계명대(대명캠퍼스) – 꽃보다 남자, 미스터 션샤인 등 촬영, “한국의 작은 하버드”라 불리는 아름다운 캠퍼스.

로컬 맛집:

막창(대구 별미) – 제일숯불막창: 아이유가 효리네 민박에서 극찬해 성지화. 위치(수성구), 콩가루 찍어 먹는 독특 맛 포인트 강조. 벽면 사인과 방송 사진 다수.

곱창전골 – 버들식당(두류공원 인근, 50년 노포): 대한민국 100대 맛집 선정, JTBC 밤도깨비에 FT아일랜드 이홍기·뉴이스트 JR 출연 화제.

떡볶이 – 중앙떡볶이(시내 전설의 분식): 국가대표 축구선수·안재욱·2PM 준케이 사인, 백종원 소개. 납작만두(서문시장, 슈가 최애 간식) 언급.

포토·전망: 앞산공원(시티뷰), 달성공원(옛 성곽/소동물원, V의 유년 추억 장소), 서문시장, 김광석 다시 그리기 길 등.

브로셔 자동 작성 프로세스:

사실 수집: 주소/영업/배경(셀럽 방문·촬영지)을 조사·정리(예: Creatrip 가이드). Google Sheet 또는 “Daegu info” 컨텍스트 파일에 구조화 저장. 예: 제일숯불막창 주소·운영시간(17:00–02:30), 계명대 촬영 타이틀 등.

AI 카피라이팅: 섹션별 명료·매혹적 문구 생성(‘반드시 가보고/먹고 싶게’).

프롬프트 예: “제일숯불막창 브로셔 카피 2–3문장. 아이유·효리네 민박 언급으로 후킹, 맛의 묘사 1요소, 방문 유도 멘트.”

결과 예: “제일숯불막창 — 효리네 민박에서 아이유도 반한 막창! 숯불 향에 고소한 콩가루를 푹 찍어 한입 베어물면, 왜 대구의 자존심인지 바로 알게 됩니다. 벽 가득한 셀럽 사인 앞에서 인증샷 남기고, 대구의 밤을 제대로 맛보세요.”

언어/톤: 일본인 대상이면 일본어로 직접 생성 또는 한→일 번역( GPT 또는 Google Translate API ). 필요 시 한·일 병기도 가능. 일본어는 공손하면서도 또렷한 감탄·권유 표현(“ぜひ味わってみてください！”)을 사용.

레이아웃 & 디자인 자동화:

Google Slides/Docs 템플릿: 8~10페이지 슬라이드 템플릿(제목/본문/이미지 자리표시자). Make에서 Google Slides API(HTTP)로 복제→플레이스홀더 치환→PDF 내보내기. Docs도 가능하나 이미지 배치는 Slides가 유리.

HTML→PDF: 에이전트가 HTML 생성, HTML-to-PDF API(PDFShift 등)로 변환(정밀 레이아웃 가능).

Canva/Adobe Express: 템플릿 필드 치환형 API도 가능하나 진입장벽 고려. 1차는 Slides 권장.

이미지: 각 장소 고해상도 사진을 수집(공식/스톡/인스타 큐레이션). 초기엔 수동 큐레이션이 품질에 유리, 이후 API 자동화로 대체 가능. 에이전트가 추천 이미지를 삽입하고, 필요 시 사람이 최종 교정.

자동 vs 반자동: 카피·이미지 추천·배치까지 반자동으로 완성, 최종 인쇄 전 1회 시각 검수로 품질 확보(인쇄물 특성). 시즌/이슈 갱신은 해당 페이지만 재생성.

온라인 리서치 출처: 공식 사이트 대신 블로그·커뮤니티 위주로 풍부한 맥락을 확보:

Creatrip: 셀럽·촬영지·사인·에피소드 정보(아이유/효리네, 방송 출연 식당, 중앙떡볶이 사인 등).

Instagram: 실제 음식/분위기·해시태그 데이터(#제일숯불막창 등).

일본인 여행 포럼: K-pop 성지·식도락 선호 확인.

대구의 BTS 슈가 벽화. 일본인 ARMY들의 필수 코스이므로, 브로셔의 “Daegu for ARMYs” 섹션에 대표 포토스팟으로 수록합니다.

이 설계는 Make.com 최신 UI에서 AI Agent(시스템 프롬프트·컨텍스트·모듈/시나리오·MCP) 를 올바르게 구성하고, 커뮤니티 검증 스토리로 영상 제작을 자동화하며, 일본 관광객 맞춤 대구 브로셔를 빠르게 제작할 수 있게 합니다. 글로벌 도구(ChatGPT, Google)와 로컬 인사이트(한국/일본 커뮤니티 트렌드) 를 결합해, 지시 한 번으로 즉시 생성·보고가 가능하도록 설계했습니다. 결과적으로 Naruu는 콘텐츠 생산 주기 단축과 맞춤 마케팅 자산(브로셔) 를 자동 확보할 수 있습니다.
