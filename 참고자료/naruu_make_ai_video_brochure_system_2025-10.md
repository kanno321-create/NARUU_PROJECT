\# 동영상 콘텐츠 \& 대구 브로셔 자동화를 위한 AI 에이전트 설계 (2025년 10월)



\## 0) 목표

\- Naruu 인스타그램/유튜브 쇼츠 채널을 위한 \*\*트렌드 스토리 영상\*\* 대량 제작 자동화

\- 일본인 여성(특히 주부·한일국제결혼) 타깃 \*\*공감형/유머형 에피소드\*\* 발굴→대본→TTS→이미지→영상→승인→게시

\- 일본인 관광객용 \*\*대구 하이라이트 브로셔(10p 이하)\*\* 반자동 제작(텍스트/이미지/레이아웃/PDF)

\## 1) Make.com AI Agent 최신 UI(2025-10) 설정 개요

\- \*\*AI 제공자/모델\*\*: OpenAI GPT-4/ChatGPT-5 연결(에이전트 추론 코어)

\- \*\*System Prompt\*\*: 역할/목적/제약을 명확히(대본·브로셔 톤, 도구 사용 원칙)

\- \*\*Context(지식파일)\*\*: 스타일 가이드, 예시 스토리, 대구 정보 등(최대 20개 파일, 각 ≤20MB)

\- \*\*MCP(선택)\*\*: 외부 검색/DB 도구가 있으면 연결 후 개별 툴 토글

\- \*\*Tools\*\*:

&nbsp; - \*\*모듈 툴\*\*: 단일 스텝(예: Google Sheets 검색, OpenAI Chat, Google TTS, Image Gen)

&nbsp; - \*\*시나리오 툴\*\*: 다중 스텝(입·출력 필드 정의, On-Demand 실행)

\- \*\*테스트 \& 튜닝\*\*: Testing chat로 프롬프트/툴 설명 점검, 과도한 시스템프롬프트 지양

\## 2) 자동화 동영상 제작 파이프라인

\### 2.1 입력/아이디어 소싱

\- \*\*소스\*\*: 한국·일본 커뮤니티(주부/국제결혼/유머), YouTube/Instagram 트렌드, 팀 수동 큐레이션

\- \*\*DB\*\*: Google Sheets(`Story\_ID, Title, Summary, SourceLink, PopularityScore, Status`)

\- \*\*선정 로직\*\*: 미사용 \& Popularity 상위 → 1건 선택



\### 2.2 스크립트 생성(Text)

\- \*\*OpenAI Chat(모듈 툴)\*\*: 1인칭, 주부공감 유머 톤, 낭독 30–60초 또는 3–4분 버전

\- \*\*스타일 규칙\*\*(요지): 오프닝 훅 → 갈등/반전 → 공감 포인트 → CTA(Naruu/대구 노출)



\### 2.3 TTS(오디오)

\- \*\*Google Cloud TTS(권장)\*\* 또는 Naver Clova/Polly

\- \*\*보이스\*\*: 한국어 여성(기본), 필요 시 일본어 버전 병행

\- \*\*출력\*\*: MP3 → Google Drive 업로드(URL 확보)



\### 2.4 이미지

\- \*\*AI 이미지\*\*: DALL·E 3 / Stable Diffusion(장면 설명 → 일러스트)

\- \*\*스톡\*\*: Unsplash/Pexels API(대체/보강)

\- \*\*권장\*\*: 커버 1–2장(중간 컷 전환)



\### 2.5 영상 합성

\- \*\*CloudConvert API\*\* 또는 \*\*서버리스 FFmpeg\*\*(정지 이미지 + 오디오 → MP4, 9:16/60초 이내)

\- \*\*인스타 릴 규격 유의\*\*: MP4/MOV, moov atom front, H.264/HEVC, 23–60fps, 4:2:0, AAC 48kHz

\- \*\*산출\*\*: MP4 → Google Drive 저장 / YouTube 업로드(비공개) → 검토 링크 리포트



\### 2.6 승인/수정/게시

\- \*\*승인\*\*: 사람 확인 → 필요 시 스크립트/보이스만 재생성 후 재합성

\- \*\*게시\*\*: YouTube(제목/설명/해시태그 자동), Instagram Reels(규격·권한 충족 시)



\### 2.7 오류/예외 핸들링

\- \*\*인스타 에러 2207026\*\*: 인코딩 불일치 → moov front/코덱/샘플레이트 재확인, FFmpeg 재래핑

\- \*\*URL 미디어 미확보\*\*: 실제 파일 URL 확보(HTTP Get file) 후 전달

\- \*\*location/page ID 오류\*\*: Graph API 지정 스키마 준수

\## 3) 트렌딩 스토리 소싱(주부 유머 \& 한일국제결혼)

\- \*\*한국\*\*: FM코리아/루리웹/네이트판/카페/유머 큐레이션 페이지(조회·추천·댓글 지표 상위)

\- \*\*일본\*\*: Ameba/Hatena/YouTube(日韓カップル)/Twitter·Threads/여행 커뮤니티

\- \*\*스토리뱅크 운영\*\*: 핵심 플롯·인기 이유(훅/펀치/교훈) 기록 → 에이전트가 대본화

\- \*\*자동 모니터\*\*: 주 1회 RSS/검색/키워드 필터(반자동도 OK)

\- \*\*톤 가이드\*\*: 유머는 가볍게/배려, 감동은 공감/따뜻함, 과장·고정관념 금지

\## 4) 대구 브로셔(10p 이하) 자동/반자동 제작

\### 4.1 콘텐츠 구성(일본인 타깃)

\- \*\*K-POP 랜드마크\*\*: BTS 슈가/브이 벽화·아트월(포토스팟)

\- \*\*드라마 촬영지\*\*: 계명대(대명캠) — 꽃보다 남자/미스터션샤인 등

\- \*\*맛집\*\*:

&nbsp; - 제일숯불막창(아이유 극찬, 콩가루 디핑)

&nbsp; - 버들식당(곱창전골, 50년 노포, 예능 출연)

&nbsp; - 중앙떡볶이(사인 다수, 납작만두·서문시장 연계)

\- \*\*뷰/산책\*\*: 앞산공원/달성공원/서문시장/김광석길



\### 4.2 작성/번역/톤

\- \*\*AI 카피\*\*: 각 장소 2–3문장(후킹·감각 묘사·방문 유도)

\- \*\*언어\*\*: 일본어 단독 또는 한·일 병기

\- \*\*표현\*\*: 밝고 상냥, “꼭 가보고 싶다/먹어보고 싶다” 감정 유발



\### 4.3 레이아웃/출판

\- \*\*Google Slides 템플릿\*\*: 제목/본문/이미지 자리표시자 → API 치환 → PDF export

\- \*\*대안\*\*: HTML→PDF 변환(API), Canva/Adobe Express(필드 치환형)

\- \*\*이미지\*\*: 고해상도(공식/스톡/인스타 큐레이션), 저작권 주의

\- \*\*프로세스\*\*: 자동(초안) → 1회 시각 검수(인쇄용) → 최종 PDF

\## 5) 오케스트레이션 설계(두 경로)

\- \*\*A안(단일 시나리오 툴)\*\*: `StoryVideoGenerator(Idea\_ID)` 한 방에 전체 수행

\- \*\*B안(다중 툴·에이전틱)\*\*: `getIdea`→`writeScript`→`makeAudio`→`makeImage`→`assembleVideo` → 재사용성·가시성↑

\- \*\*전략\*\*: 초기 A안으로 빠른 가동 → 성숙 후 B안으로 분해·최적화

\## 6) 도구 요약(한국 실사용 선호 반영)

\- \*\*Make.com\*\*: Agent·모듈·시나리오·스케줄/온디맨드

\- \*\*OpenAI(ChatGPT-4/5)\*\*: 대본/브로셔/번역/요약

\- \*\*Google Sheets/Drive\*\*: DB·자산 저장

\- \*\*Google Cloud TTS / Naver Clova / Polly\*\*: 더빙

\- \*\*DALL·E / Stable Diffusion / Unsplash·Pexels\*\*: 이미지

\- \*\*CloudConvert / FFmpeg(서버리스)\*\*: 영상 합성

\- \*\*YouTube / Instagram\*\*: 배포(규격·권한 고려)

\## 7) 운영 체크리스트(10분 임포트)

1\) Make 로그인 → \*\*Scenarios → Import blueprint\*\*(필요 시)  

2\) \*\*Connections\*\*: OpenAI/Google/YouTube/CloudConvert 등 연결  

3\) \*\*변수 매핑\*\*: 제목/설명/파일URL/태그  

4\) \*\*Run once\*\* 테스트 → \*\*Schedule\*\* On / Agent Tool 활성화  

5\) 실패로그 모니터 → 프롬프트·툴 설명 튜닝
